## 二、六步归档流程

每次收到新的跑团聊天记录，**必须**依次执行：

| 步骤 | 写入文件 | 核心任务 |
|---|---|---|
| 0 | `00_当前局势.md` | 🔴 先扫30秒——了解当前位置/活跃线索/紧急待办 |
| 1 | `01_线索板.md` | 提取新线索，编号归档（活跃版） |
| 2 | `02_人物关系.md` | 提取新NPC，更新关系图谱 |
| 3 | `03_时间线.md` | 提取事件，按时间排列 |
| 4 | `04_行动日志.md` | 记录当前场景行动+骰子结果 |
| 5 | `05_推测与假设.md` | 交叉关联分析（严格分三级） |
| 6 | `06_待办事项.md` | 更新下一步行动建议 |

**新线索归档规则：**
- ✅ 已确认 + 不再需追查 → 同步写入 `01a_事实库.md`
- 🟡 待验证 / 🔴 活跃追查 → 只写入 `01_线索板.md`
- 场景完成后 → 将 `04_行动日志.md` 内容移到 `04a_过往日志.md`，重置为当前场景

### 每条线索必须标注

- **【来源】**：KP叙述 / 某角色发言 / 骰子结果 / 场外信息
- **【可靠性】**：见第四节证据标准
- **【关联编号】**：引用已有关联线索编号

## 三、文件结构定义

```
跑团日志/
├── 00_当前局势.md             ← 🔴 每轮必读 — 一页速览仪表盘
├── README.md                 ← 总索引 + 角色速查
├── 01_线索板.md              ← 活跃追查中的线索 + 待验证
├── 01a_事实库.md             ← 已确认基石事实（分类归档）
├── 02_人物关系.md            ← NPC图谱，立场/动机/关联
├── 03_时间线.md              ← 按日期的案件时间轴
├── 04_行动日志.md            ← 当前场景日志（保持轻量）
├── 04a_过往日志.md           ← 已完成场景归档
├── 05_推测与假设.md          ← 推理（严格分级：已证实/需证实/仅推测）
├── 06_待办事项.md            ← 下一步调查方向（按优先级）
└── 07_跑团规范手册.md        ← 本文档
```

### 分层目的

| 层级 | 文件 | 设计意图 |
|---|---|---|
| 🔴 即时 | `00_当前局势.md` | 每轮对话前30秒扫一遍 |
| 🟡 活跃 | `01_线索板.md` / `04_行动日志.md` / `06_待办事项.md` | 调查中需要翻阅 |
| 🟢 归档 | `01a_事实库.md` / `04a_过往日志.md` | 深挖/复盘/结案时才用 |
| ⚪ 稳定 | `02_人物关系.md` / `03_时间线.md` / `05_推测与假设.md` | 定期更新但不会线性膨胀 |

## 四、证据标准（铁律）

### 4.1 线索可靠性（用于`01_线索板.md`）

| 标注 | 含义 | 示例 |
|---|---|---|
| ✅ **已确认** | KP叙述/骰子结果/多源交叉验证 | "瞳孔对光反射消失——藤堂咲手电筒+KP确认" |
| 🟡 **待验证** | 单一角色口述，未交叉验证 | "其他失明者'好像都是香港人'——玛莎" |
| 🔴 **紧急关注** | 重大异常，即使已确认也标红 | "华莱士教授失联" |
| ℹ️ **背景信息** | 场外元信息/玩家行为记录 | "林芷信用检定失败75/55" |
| ⚠️ **低可靠** | 来源可信度存疑 | "KP揶揄朗格不是真侦探" |

### 4.2 推测可靠性（用于`05_推测与假设.md`）

| 标签 | 含义 | 能否作为决策依据 |
|---|---|---|
| 🟢 **已证实** | KP原话确认/多源交叉验证一致 | ✅ 可作为分析基底 |
| 🟡 **需证实** | 基于已证事实的合逻辑推断 | ⚠️ 需验证后使用 |
| 🔴 **仅推测** | 无直接事实支撑，纯猜测 | ❌ 不可作为决策依据 |

**铁律：** 缺乏KP确认、缺乏多源验证 = 降低一级。永远不把🔴仅推测当作决策依据。

## 六、编写规范速查卡

### 禁止

| 禁止 | 原因 |
|---|---|
| ❌ 未标注来源的线索 | 后期复查无法追溯 |
| ❌ 推测与事实混放 | 混淆是最大的敌人 |
| ❌ 删除已被推翻的假设 | 标记"已排除"而非删除 |
| ❌ 用OOC视角主导现场RP建议 | 先走刑侦路线 |

### 必须

| 必须 | 原因 |
|---|---|
| ✅ 每条新线索给唯一编号 | 便于交叉引用 |
| ✅ 每个假设标注状态标签（🟢/🟡/🔴） | 防混淆铁律 |
| ✅ 推测升级后更新标签 | "需证实→已证实"及时标记 |
| ✅ 线索与假设互引 | 线索板引用假设编号，假设引用线索编号 |
| ✅ 骰子结果即时记录 | 检定失败/成功的信息质量不同 |
| 🔴 新 | **线索确认后→同步归档到01a_事实库。场景完成后→移动到04a。保持活跃文件轻量。** |
