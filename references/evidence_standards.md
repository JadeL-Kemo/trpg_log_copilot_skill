# 证据标准 — 双维溯源体系

> **v1.5.1**：本文件定义理论框架。实际数据文件中，此框架被扁平化为表格的"来源"和"可靠性"两列。
> 详见下方"实际格式映射"。

## 实际格式映射（数据文件中使用的格式）

| 数据文件列 | 取值示例 | 对应理论标签 |
|------|------|------|
| **来源** | `KP旁白` / `KP场景` | 场内 |
| | `角色名`（如"玛莎""赵婉清"） | 场内 |
| | `骰子`（含技能名+值） | 检定 |
| | `场外`（标注玩家名） | 场外 |
| **可靠性** | `✅ 已确认` | 高/确定 |
| | `🟡 待验证` / `🟡 叙事提示` / `🟡 主观判断` | 中 |
| | `🔴 紧急` | 高（紧急关注） |
| | `⚠️ 低可靠` / `⚠️ OOC直觉` | 低 |
| | `ℹ️ 背景` | 背景信息 |

## 理论框架：来源标签

| 标签 | 含义 | 示例 |
|------|------|------|
| `场内` | 游戏内NPC/场景/道具直接给出的信息 | NPC说"<具体信息>"、场景描述 |
| `检定` | 骰子/技能检定结果 | <技能>(<值>)成功/失败 → <发现或未发现的内容> |
| `场外` | KP闲聊/暗示/环境描述（有真相但无角色获取渠道） | KP暗示或闲聊中透露的<信息> |
| `推测` | 玩家/副官推理（既无证据也无真相确认） | "<基于线索的推理>" |
| `铁证` | 参考资料+剧情佐证+人证+KP明示 四者齐备 | <文献/资料>+NPC证词+KP确认 |

## 确信度标签

| 标签 | 适用来源 | 含义 |
|------|----------|------|
| `高` | 场内/检定/场外/推测 | 有可靠信息来源或强逻辑支撑 |
| `中` | 场内/检定/场外/推测 | 有一定依据但存在不确定因素 |
| `低` | 场内/检定/场外/推测 | 直觉或弱线索 |
| `确定` | 铁证 | 无可争议（铁证固定值） |

## 来源跨越规则

**来源标签不可越级变更。**

| 从 | 到 | 条件 |
|----|----|------|
| 推测 → 场内 | ✅ | 需要新证据支撑 |
| 推测 → 铁证 | ❌ | 不能直接从推测跳到铁证——必须先成为场内确认信息 |
| 场外 → 场内 | ✅ | 游戏内角色获取到了对应信息 |
| 检定 → 场内 | ✅ | 检定结果被其他来源证实 |
| 场内 → 铁证 | ✅ | 积累足够证据 + KP确认 |

确信度可以升降（如新证据推翻了旧认知），但来源标签只能在获取新类型信息时变更。

## 防超游防火墙

| 层 | 位置 | 可带入角色？ |
|----|------|------------|
| 场内已确认 | `01a_事实库.md`, `04_行动日志.md` | ✅ |
| 场内待验证 | `01_线索板.md` | ✅（标注来源确信度） |
| 场外信息 | `05_推测与假设.md` 场外区 | ❌ 禁止带入角色知识 |
| 纯推测 | `05_推测与假设.md` 推测区 | ⚠️ 角色可以"觉得不对劲"，但不能"知道" |
