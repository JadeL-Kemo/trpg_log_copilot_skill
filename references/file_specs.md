# 文件体系详细规范

> 每个文件的格式定义、字段说明、编写规则和示例

---

## 文件分层设计

| 层级 | 文件 | 加载时机 | 说明 |
|------|------|----------|------|
| 🔴 即时 | `00_当前局势.md` | **每轮必读，≤30秒** | 一页速览仪表盘 |
| 🟡 活跃 | `01_线索板` / `04_行动日志` / `06_待办事项` | 调查进行中 | 频繁翻阅 |
| 🟢 归档 | `01a_事实库` / `04a_过往日志` | 深挖/复盘/结案时 | 平时不翻 |
| ⚪ 稳定 | `02_人物关系` / `03_时间线` / `05_推测与假设` | 定期更新 | 不会线性膨胀 |

## `01_线索板.md` — 活跃线索

### 线索编号体系

| 前缀 | 含义 | 示例 |
|------|------|------|
| `CL-XXX` | 核心线索（Core Line） | CL-001 |
| `UV-XXX` | 待验证项（Unverified） | UV-001 |
| `SL-XXX` | 支线线索（Side Line） | SL-001 |
| `TC-XXX` | 队友能力（Teammate Cap） | TC-001 |
| `PL-XXX` | 地点/物证（Place/Loot） | PL-001 |

### 每条线索的格式

```markdown
| 编号 | 内容 | 来源 | 可靠性 | 关联 |
|------|------|------|--------|------|
| CL-XXX | [线索内容] | [KP叙述/某人发言/骰子结果] | [标注] | CL-YYY, UV-ZZZ |
```

### 可靠性标注

| 标注 | 含义 |
|------|------|
| ✅ 已确认 | KP 叙述 / 骰子结果 / 多源交叉验证 |
| 🟡 待验证 | 单一角色口述，未交叉验证 |
| 🔴 紧急关注 | 重大异常，即使已确认也标红 |
| ℹ️ 背景 | 场外元信息 / 玩家行为记录 |
| ⚠️ 低可靠 | 来源可信度存疑（含 OOC） |

## `05_推测与假设.md` — 推理分级

### 推测可靠性标签（铁律）

| 标签 | 含义 | 能否作为决策依据 |
|------|------|-----------------|
| 🟢 已证实 | KP 原话确认 / 多源交叉验证 | ✅ 可作为分析基底 |
| 🟡 需证实 | 基于已证事实的合逻辑推断 | ⚠️ 需验证后使用 |
| 🔴 仅推测 | 无直接事实支撑，纯猜测 | ❌ 不可作为决策依据 |

### 每个假设的格式

```markdown
### H-XXX: [假设标题] [标签]
- **基础事实：** [引用的线索编号]
- **确认来源：** [KP确认/多源验证]
- **推断：** [推理内容]
- **验证方向：** [如何验证]
```

### 铁律

- 🔴 仅推测 **永远不可作为决策依据**
- 缺乏 KP 确认 → 降一级
- 已排除的假设标记"已排除"，**禁止删除**

## `07_跑团规范手册.md` — 自包含迁移文档

参见 `references/evidence_standards.md` 获取证据标准详细规则。

手册应包含：
1. 角色与身份（玩家角色 + AI 助手角色 + 回复格式要求）
2. 六步归档流程
3. 文件结构定义
4. 证据标准（铁律）
5. 角色技能速查
6. 当前案件概要
7. 编写规范速查卡（禁止/必须）
8. 切换平台加载方式（System Prompt 模板）
